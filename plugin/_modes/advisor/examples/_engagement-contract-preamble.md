# Engagement contract — shared preamble for all example deliverables

Every deliverable template in this folder operates under the same engagement contract. Read this once; each example file references it instead of repeating it.

## What every example USES (artifact inputs read at runtime)

- **Brand kit resolution**: the active kit at `template_assets/branding/<active-slug>/` — defaults to `_default` (the bundled neutral placeholder seed kit per `brand-guidelines.md`); overrides resolve when an engagement-config sets `deck.brand: <org-slug>` and a kit exists at `template_assets/branding/<org-slug>/`. Theme JSONs (palette, typography, footnotes) deep-merge over `_default`; masters and logos full-file-replace. Full mechanics: `references/brand-kit-protocol.md` § Per-org override mechanism. Resolution order between backend, local cache, and bundle: `references/library-resolution.md`.
- **Engagement config keys honored** (per `references/engagement-config-template.md`): `deck.brand` (which kit), `deck.production_mode` (interactive | silent | hybrid), `deck.artifacts` ([pptx, xlsx, csv]), `deck.slide_count_by_audience.<archetype>`, `deck.required_appendix`, `deck.decision_ask_pattern`, `audience.archetypes[]` (matched by id from the narrative frame), `benchmark.default_percentiles` ([10, 25, 50, 75, 90] is the load-bearing default — never silently subset), `benchmark.scope_provinces` (drives per-province roll-ups when 2+), `costing.target_percentile`, `costing.roll_up_factor`, `costing.payroll_burden_pct` (per-province; required on costing-slide caveats), `org.name` / `org.industry` (header strings). When `deck.brand` is unset, defaults to `_default` (neutral placeholder); when `audience.archetypes` is unset, falls back to skill defaults from `references/engagement-config-template.md` § audience.
- **Backend reads** (per `references/persistence-and-ledger.md`): the active brand kit via `brand_get_kit` / `brand_get_file` / `brand_list_files`, any per-org `voice-and-design-rules.md` override shipped with the kit, the engagement's `engagement-state` body via `engagement_get` (for prior-section context in interactive Phase 6b), and prior cycles + decision log via `engagement_get_master.cycles[]` for `prior_engagement_refs` continuity. Local `$STATE_ROOT` serves as read cache only on transport failure (D1).
- **Verified-source `tool_calls[]` array**: every numeric claim cited in the deliverable must trace to an entry in the engagement-state's `tool_calls[]` array per `references/tools-available.md` § Verified-source discipline. The 11-tag vocabulary (`[statcan-wage]`, `[live-postings]`, `[cba]`, `[indeed-company]`, `[econometric]`, `[statutory]`, `[market-data]`, `[survey-house]`, `[user-provided-cba]`, `[professional-judgment]`, `[assumption]`) gates source citations; auto-downgrade fires on any tag without a matching `tool_calls[]` entry.

## What every example PRODUCES (END artifacts)

- **Primary deliverable**: a file artifact (`.pptx`, `.md`, `.xlsx`, or `.csv` per the example's track) — produced via the file artifact primitive in claude.ai, NOT chat-only output. Filename pattern includes the engagement slug + ISO date.
- **Optional companions** when `deck.artifacts` requests them: `cost-scenarios.xlsx` (Phase 6 alongside PPTX) and `market-data.csv` (Phase 6 alongside PPTX). See `references/artifact-generation.md` § cost-scenarios.xlsx and § market-data.csv for sheet/column structure.
- **Structural state written** (schema-state writes are MCP-only, D2): the `engagement-state` body via `engagement_put` (Phase 7 close-time write sequence) — including the `tool_calls[]` audit array and `selection_log[]` entries for any A/B picks made during Phase 6b — plus a stub cycle record via `engagement_put_cycle` with `outcome_observed_*` fields null (outcome writeback lands later via `/ledger update`). All writes pass `expected_version`; on write failure, escalate/halt (never write-local). See `references/persistence-and-ledger.md` § Command mapping.
- **Chat preview**: a one-screen summary of the file artifact (headline + key numbers + decision-ask) so the user can react without opening the file. Never substitute the chat preview for the file artifact.

## Brand enforcement (per-example)

Each example's `## Brand enforcement` section names which active brand kit governs and which structural rules apply across kits. The `_default` kit ships neutral placeholder content per `brand-guidelines.md`; per-org kits override the kit-specific items (palette, typography, logo files, brand framing) and may ship their own `voice-and-design-rules.md`. The structural items in the compliance checklist (no stock people imagery, ≥0.5" margins, one message per slide, single-colour icons paired with text, voice direct and jargon-calibrated) apply across kits regardless of which kit is active.

Run the active kit's compliance checklist as part of Phase 7 QA. For the `_default` kit, that's the checklist at the bottom of `brand-guidelines.md`; for a per-org kit, follow that kit's `voice-and-design-rules.md` checklist.
