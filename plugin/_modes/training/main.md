## When to use this skill

- **Source extraction** — ingest a comp-team-transformer transformation-brief, a policy doc, a guideline PDF, or a prior PPTX into a canonical message-map with audience+depth tagging
- **Per-audience render** — generate a training bundle (PPTX deck + facilitator guide + interactive blocks) for one or more of: employees, managers, HRBPs, execs
- **Cascade derivation** — derive a 30-min team-meeting variant from the manager training deck so the manager runs the deck WITH their team, not AT them
- **Brand kit setup** — read or write the engagement brand kit shared across comp-advisor, comp-team-transformer, comp-training-designer
- **Strategic deliberation** — multi-perspective reasoning on contested per-audience message-map decisions (HRBP, comp-manager, audience-recipient, hris-tooling, change-management)
- **Engagement init** — first-time setup of `engagement-training-config.yaml` with audiences, depth-layer mapping, source pointers, brand-kit pointer
- **Mid-cycle checkpoint / resume** — save state and resume a multi-session ingest → generate → cascade pipeline without re-walking covered ground

## Output format

Working artifacts are markdown (message-map.yaml, facilitator guides, interactive blocks). Shareable training decks are PowerPoint (.pptx) for `/generate` (required, per audience) and `/cascade` (required, manager team-meeting variant). Before slide construction, read the pptx skill at `/mnt/skills/public/pptx/SKILL.md`, `$ASSET_ROOT/_modes/training/references/template-master.md`, and `$ASSET_ROOT/_modes/training/references/brand-kit-protocol.md`. The brand template is regenerated fresh from `template_assets/branding/_default/build_template.js` (or the per-org override at `branding/<org_slug>/` in local `$STATE_ROOT`) — there is no bundled binary PPTX. Default branding from `branding/_default`; switch via engagement-training-config `voice.brand.org_slug` for an org kit. Every deck is built new from the regenerated template. Three production checks before any write: brand kit applied, redaction pass complete, audience tag present in frontmatter. Every rendered deck additionally carries `delivery_target` metadata on the cover slide and in deck frontmatter.

## Tools available

This skill uses the **`market` MCP backend** (OAuth identity → org membership) as the persistence backend for schema state (engagement configs, message-maps, cycle records, checkpoints, decisions). Non-schema artifacts (rendered decks, facilitator guides, council scratch, cost-log) write to local `$STATE_ROOT`. Persona libraries and vocabulary/glossary are LOCAL — not stored in the backend. Council and reasoning are single-context — no external MCP calls during deliberation. Full tool inventory and tool-selection priority table live in `$ASSET_ROOT/_modes/training/references/tools-available.md` (mirrored from github.com/david-deji/compensation-advisor — manual sync on change). The v2 persistence contract lives in `$ASSET_ROOT/_modes/training/references/persistence-and-ledger.md`. Read each whenever a phase needs to pick a tool or verify a write path.

**Configuring persistence**: schema state persists automatically via the `market` MCP backend once the user is authenticated (OAuth). Run `/init` to create or load the engagement config; the backend is always reachable post-auth. On transport failure, the skill falls back to the local `$STATE_ROOT` read cache (D1) — schema writes escalate/halt. See `$ASSET_ROOT/_modes/training/references/persistence-and-ledger.md`.

### Critical orchestration rules

- **Two-register output discipline.** Operational output and deliverable output use different registers and must not blend. **Operational** = which step ran, what got written, which probe succeeded, which assumption was checked, which file was loaded — terse or silent. Surface only steps that produce a user decision or visible content. Never narrate no-op steps ("Step N — nothing to inherit, skipping" → just skip silently and proceed). **Deliverable** = message-map synthesis, facilitator guide prose, slide narratives, council synthesis, audience-facing training text — full prose, narrative arc, audience calibration. The skill's playbook tone (facilitator / instructional designer) belongs in deliverables; operational steps run quietly. Mixing the registers — applying deliverable register to operational steps — produces ceremony that makes a 5-minute init feel like 30 minutes. Walk the protocol; do not perform it.
- **Verify-before-assert on factual claims.** Any claim involving a present-day rate, statute, role-holder, recent policy change, vendor capability, or fact about an external system requires a `web_search` (or, for tools, a `tool_search`) BEFORE assertion. Training-data priors are not acceptable for any value that can change between sessions. When you have not verified, the correct phrasing is "I haven't checked — let me verify"; never "X is the case, as of last year." Inventing capability or fact categories to soften an earlier wrong claim is itself an error.
- **Artifact philosophy.** Every mode produces a durable consumption artifact. Markdown is the END artifact for working documents (message-map.yaml, facilitator guides, interactive blocks). PPTX is the END artifact for delivered training material on `/generate` (required, per audience) and `/cascade` (required). Council always produces `council-state.yaml` (in-mode: alongside the rendered audience; standalone: as the END deliverable + optional decision-memo.md). Utility command END artifacts: `/init` → `engagement-training-configs/<slug>.yaml`; `/brand` → engagement-root `branding/<org-slug>/`; `/checkpoint` → `checkpoint.yaml`. Only `/help` is chat-only.
- **Redaction is a hard rule.** Phase 0 scans every pasted/loaded input for banned patterns (raw names, raw salary figures, raw headcounts, personal contact info). On detection: refuse to proceed, surface the warning, instruct re-paste with redaction. Pre-write scan repeats at every artifact write. Required transformations: salary → band/percentile, headcount → size_band, names → role + function, company name → kept in private folder, replaced with `<COMPANY>` only when artifact tagged `audience: external`. Does NOT relax under user pressure. Full rules in `$ASSET_ROOT/_modes/training/references/redaction-rules.md`.
- **Differentiated depth is enforced.** Per `$ASSET_ROOT/_modes/training/references/audience-depth-rules.md`: on the same fact (`msg-XXX` in message-map.yaml), HRBP depth ≥ manager depth ≥ employee depth (when all three are non-null). Execs see different cuts (tradeoffs / budget / governance) — depth-4 framing, not deeper-of-the-same. Schema validation enforces this at `/ingest` Workshop synthesis. `/generate` refuses to render an audience that would violate the constraint.
- **Delivery-target metadata, no cycle gating.** Every rendered deck (employees / managers / managers-cascade-kit / hrbps / execs) carries `delivery_target` metadata on the cover slide and in deck frontmatter. The metadata is computed from `cycle.current_stage` + operator-supplied `delivery_target_week_offset`. **No gating** — training is often intentionally delivered in live/prep windows (cascade in week 0, exec ratification in prep, HRBP cycle-mechanics training in prep). Soft warning fires only if `delivery_target` is in the past. Rule in `$ASSET_ROOT/_modes/training/references/cycle-awareness.md`.
- **Council runs single-context.** Personas are sequential voice blocks within one response, not parallel subagent dispatch. Never draft the synthesis block until every persona block is complete. Auto-fires on `/generate` per-audience message-map slice locks per the auto-fire matrix in `$ASSET_ROOT/_modes/training/references/council-mode.md`. No weighted scoring in v1 — categorical depth votes only. `/cascade` does NOT auto-fire (mechanical derivation, no contested decisions).
- **Conversational interview discipline.** `/ingest` and `/generate` audience-design interview run as single continuous conversations: the skill interviews the operator directly using pros/cons option tables and Mom Test follow-ups. The skill asks one question at a time; the operator answers. Pattern-trigger table in `meta-protocol.md` governs follow-up routing. No separate Prep / Whisper / Workshop modes — the interview is one arc from scripted opening to reflect-and-verify to scripted close.
- **Each checkpoint blocks the next phase until the user explicitly confirms.** No silent advance. `/checkpoint` writes are synchronous; user reviews and proceeds.
- **Persistence discipline — backend for schema, local for artifacts, binaries as chat-download.** Schema state (engagement config, message-map, cycle records, checkpoint, decisions) writes through the `market` MCP backend via `engagement_put_section` / `engagement_put` / `engagement_append_decision`. Non-schema artifacts (facilitator guides, interactive blocks, council-state.yaml) write to local `$STATE_ROOT`. **Binary artifacts (per-audience PPTX decks for /generate, /cascade) NEVER go through any backend** — they deliver as chat-download artifacts and the user files them into their own storage (e.g., via Drive desktop sync, which auto-uploads outside any LLM context). The skill records the intended local path in metadata for provenance; bytes are never written to any backend. This is a hard architectural rule — no toggle, no "upload anyway" path. Co-dependent local artifacts use the staged-write pattern (write to `_pending/`, move on success) so partial failure is orphan-detectable. Backend writes use optimistic concurrency (`expected_version`); on stale-reject, re-read and retry. On backend transport failure, reads fall back to the local `$STATE_ROOT` cache (D1); writes escalate/halt. Full contract: `$ASSET_ROOT/_modes/training/references/persistence-and-ledger.md`.
- **Gap detection at session close.** At every `/ingest` workshop close, the skill surfaces any audience that has thin coverage: "Execs only have 2 depth-4 messages — is that intentional?" At every `/generate` audience-design close, it surfaces uncovered objections: "We don't have an objection-handling angle for [tension] — intentional?" Captured in message-map.yaml § Coverage Gaps.
- **Mirror discipline.** Six reference files are mirror copies (not symlinks): `meta-protocol.md` (canonical in the `comp-team-transformer` repo), and `persistence-and-ledger.md`, `production-and-qa.md`, `template-master.md`, `brand-kit-protocol.md`, `tools-available.md` (canonical at `github.com/david-deji/compensation-advisor`). This skill never edits them — coordinate with the canonical skill for any change, then re-copy here. The mirror header at the top of each file names the canonical source.
