## When to use this skill

- **Process discovery** — interview a colleague (or yourself) to map a comp process from scratch, or refresh an existing map; produces a structured current-state document
- **Waste / loop diagnosis** — systems-thinking pass on a discovered process: actors, system map, feedback loops, waste ledger, leverage points, Quick Wins
- **Transformation spec** — work-decomposer-style buildable spec for AI/automation augmentation of a process; categorical bands (Quick Win / Strong Candidate / Needs Groundwork / Not Ready); council deliberation on band assignments
- **Quarterly roadmap** — cross-process sequencing under cycle-fit gating; quarterly slot assignment that respects live/prep windows
- **Strategic deliberation (Council)** — multi-perspective reasoning on contested process changes (HRBP, comp-manager, comp-analyst-operator, hris-tooling, change-management); reasoning output, optional decision memo, or integrated input into `/transform` or `/roadmap`
- **Cycle discovery** — first-time mapping of the comp cycle (anchor event, stages, slack windows) during a `/discover` interview; populates `team-config.cycle.*`
- **Outcome ledger update** — log what shipped after a transformation, observed-vs-expected results, drift conditions
- **Mid-track checkpoint / resume** — save state and resume a multi-session discovery → diagnosis → transform pipeline without re-walking covered ground

## Output format

Working artifacts are markdown (current-state, diagnosis, transformation-brief, roadmap). Shareable executive decks are PowerPoint (.pptx) for `/diagnose` (optional), `/transform` (required), `/roadmap` (required). Before slide construction, read the pptx skill at `/mnt/skills/public/pptx/SKILL.md`, `$ASSET_ROOT/_modes/transformer/references/template-master.md`, and `$ASSET_ROOT/_modes/transformer/references/brand-kit-protocol.md`. The brand template is regenerated fresh from `template_assets/branding/_default/build_template.js` (or the per-org override at `branding/<org_slug>/` in the local `$STATE_ROOT`) — there is no bundled binary PPTX. Default branding from `branding/_default`; switch via team-config `voice.brand.org_slug` for an org kit. Every deck is built new from the regenerated template. Three production checks before any write: brand kit applied, redaction pass complete, audience tag present in frontmatter.

### Critical orchestration rules

- **Two-register output discipline.** Operational output and deliverable output use different registers and must not blend. **Operational** = which step ran, what got written, which probe succeeded, which assumption was checked, which file was loaded — terse or silent. Surface only steps that produce a user decision or visible content. Never narrate no-op steps ("Step N — nothing to inherit, skipping" → just skip silently and proceed). **Deliverable** = current-state synthesis, diagnosis narrative, transformation-brief prose, roadmap rationale, council synthesis, audience-facing text — full prose, narrative arc, audience calibration. The skill's playbook tone (consulting peer) belongs in deliverables; operational steps run quietly. Mixing the registers — applying deliverable register to operational steps — produces ceremony that makes a 5-minute init feel like 30 minutes. Walk the protocol; do not perform it.
- **Verify-before-assert on factual claims.** Any claim involving a present-day rate, statute, role-holder, recent settlement, vendor pricing, tool capability, or fact about an external system requires a `web_search` (or, for tools, a `tool_search`) BEFORE assertion. Training-data priors are not acceptable for any value that can change between sessions. When you have not verified, the correct phrasing is "I haven't checked — let me verify"; never "X is the case, as of last year." Inventing capability or fact categories to soften an earlier wrong claim is itself an error.
- **Artifact philosophy.** Every track produces a durable consumption artifact. Markdown is the END artifact for working documents (current-state.md, diagnosis.md, transformation-brief.md, roadmap-Qx.md). PPTX is the END artifact for executive decks on `/diagnose` (optional), `/transform` (required), `/roadmap` (required). Council always produces `council-state.yaml` (in-track: alongside the brief; standalone: as the END deliverable + optional decision-memo.md). Utility command END artifacts: `/init` → `team-configs/<slug>.yaml`; `/checkpoint` → `checkpoint.yaml`; `/ledger update` → updated `interventions-history.yaml`. Only `/help` is chat-only.
- **Redaction is a hard rule.** Phase 0 scans every pasted input for banned patterns (raw names, raw salary figures, raw headcounts, personal contact info). On detection: refuse to proceed, surface the warning, instruct re-paste with redaction. Pre-write scan repeats at every artifact write. Required transformations: salary → band/percentile, headcount → size_band, names → role + function, company name → kept in private folder, replaced with `<COMPANY>` only when artifact tagged `audience: external`. Does NOT relax under user pressure. Full rules in `$ASSET_ROOT/_modes/transformer/references/redaction-rules.md`.
- **Cycle gating.** When `cycle.current_stage` is populated, `/transform` and `/roadmap` refuse to schedule rollouts into `live` or `prep` windows. Only `slack` windows are eligible without explicit user override. Surface the conflict, offer the next slack window, accept override only with explicit acknowledgment. Rule in `$ASSET_ROOT/_modes/transformer/references/cycle-discovery-and-gating.md`.
- **Council runs single-context.** Personas are sequential voice blocks within one response, not parallel subagent dispatch. Never draft the synthesis block until every persona block is complete. Auto-fires on `/transform` (band locks) and `/roadmap` (sequence locks) per the auto-fire matrix in `$ASSET_ROOT/_modes/transformer/references/council-mode.md`. No weighted scoring in v1 — categorical bands only.
- **Conversational interview discipline.** `/discover` runs a single continuous interview in one Claude.ai conversation — the skill asks, the operator (who is also the SME being interviewed) types answers. No separate modes, no live-call copilot. The skill uses pros/cons option tables for structured choices and Mom Test behavioral questions for narrative answers. One question per turn. See `$ASSET_ROOT/_modes/transformer/references/meta-protocol.md` for the full interview contract and `$ASSET_ROOT/_modes/transformer/references/discovery-protocol.md` for the comp-domain question flow.
- **Each checkpoint blocks the next phase until the user explicitly confirms.** No silent advance. `/checkpoint` writes are synchronous; user reviews and proceeds.
- **Persistence discipline — schema backend, binary chat-download.** Schema-shaped text artifacts (team-config sections, council-state, checkpoint, decision log entries) write through `market` MCP backend tools (`engagement_put_section`, `engagement_put`, `engagement_append_decision`). Non-schema artifacts (current-state.md, diagnosis.md, transformation-brief.md, roadmap-Qx.md, council-state.yaml, interventions-history.yaml) persist locally under `$STATE_ROOT`. **Binary artifacts (PPTX decks for /diagnose, /transform, /roadmap) NEVER go through any backend** — they deliver as chat-download artifacts and the user files them into their own storage (e.g., via Drive desktop sync outside any LLM context). The skill records the intended local path in metadata for provenance; bytes are never round-tripped. This is a hard architectural rule — no toggle, no "upload anyway" path. Full contract: `references/persistence-and-ledger.md`.
- **Gap detection at session close.** At every `/discover` workshop close, the skill surfaces any sub-phase that wasn't covered: "We didn't get into <X> — is that intentional or worth a follow-up?" Captured in `current-state.md` § Coverage Gaps.
